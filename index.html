KIOKU

SNS疲れしている人に向けた、自分のためのSNS、1日で投稿が消えることことで、他人の目により触れないようにしている。

タイムラインに載せないようにすることもでき、自分の投稿一覧に溜まる仕様になっている。SNS上に、よりプライベートな投稿を投稿でき、自分の思い出『KIOKU』を残すことができる。

rails、aws、dockerを使い就職を意識して、作ったアプリ。デザインにも気を配り、基本的な機能、(投稿,更新,削除,いいね)に加え、メールで質問できる機能、など細やかに設定したアプリ。

HTML/CSS

sass、bootstrapなどフレームワークやを使って便利に、素早く実装していることがほとんどだが、実務において素早く綺麗なUIを作ることが第一だと思うので。あまりゴリゴリとcssを書かないように意識しています。

デザイン面では頭だけでは想像ができないので、AdobeXDである程度デザインを決めてから、コーディングしています。命名規則では、分かりやすいように翻訳にかけよりベターな名前をつけ、他の人に伝わりやすい名前を心がけています。

cssはすぐに汚れやすいので、ファイルを分けるようにしたり、コメントで分かりやすいように心がけています。

JavaScript

javascriptのライブラリのjQueryを使った基本的な、動きとUIの実装ができます。その他にも、非同期などを使って『KIOKU』のいいね機能を実装しました。

これからいろんなアプリを作っていく上で必須のスキルなので、Reactなど、新しい技術も導入し意欲的に学習していきたいです。

Ruby

rubyはサーバーサイド言語で1番理解しています。rubyでメソッドを書いたり、基本的なメソッドを使うためにしっかりと勉強しました。

テキストやwebテキストで意識的にrubyを勉強しています。いろんなメソッドを知りつつ、利用できるようにしています。

ruby2.7.2をよく使用しています。バージョンの関係使っていますが、より新しいバージョンを使ったり、自由に選択できるようにしっかり関係も理解していきたいです。

Rails

railsでWebアプリの作り方を学んだ。基本的な機能の仕方、テスト、gemなど、理解しています。特にdeviseはgoogle認証などいろんな使い方があるので、時間をかけて理解しました。

railsでのAPIを導入したり、より実践やいいアプリを作るための、勉強が足りてないのでそこをレベルアップしていきたいです。

rails6を使用していることが多く、これからrails7を使用していきたいです。

mysql

railsでのmysql環境構築の仕方や、railsを通してのmysqlコマンドをある程度使えます。rails以外のmysqlはこれから慣れていかないといけないです。

mysqlでは5.7を使うことが多く、新しいバージョンは扱いづらいので、徐々に新しいバージョンも使っていきたいです。
